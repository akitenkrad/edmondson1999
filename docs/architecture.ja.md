# アーキテクチャ

## ワールド状態

`TeamWorld` は以下を保持する．

- `individuals: BTreeMap<AgentId, Individual>` — ソート済みキーが正準で決定論的な活性化順序を与える．各 `Individual` は `psi` (ψ_i ∈ [0,1])・`fear` (f_i)・`ivt` (θ_i，暗黙の発言理論の強度)・`private_concern` (c_i)・transient フラグ `voiced_last` / `retaliated_last` を持つ．
- `teams: BTreeMap<TeamId, Team>` — 各 `Team` はソート済み `members`・コンテクスト `support` (s_k) と `coaching` (c_k)・集約 `psi_bar` (ψ̄_k)・`learning` (L_k)・`performance` (Π_k)・`efficacy` (η_k) を持つ．
- `network: SocialNetwork` — **チーム内辺のみ** を保持する単一の `socsim-net` グラフ (各チームを独立に Watts–Strogatz で配線．チーム横断辺は存在しない)．
- transient なチーム別カウンタ `voice_count_by_team` / `help_count_by_team` / `error_talk_by_team`．毎ステップ開始時にクリアする．

## 6 フェーズループ上の 6 メカニズム

| # | メカニズム | フェーズ | 役割 |
|---|-----------|---------|------|
| 1 | `context_support_update` | Environment | transient カウンタをクリアし，各チームの支援 `s_k` / コーチング `c_k` を目標値へ平均回帰させる (任意の外生ショック)． |
| 2 | `voice_decision_rule` / `voice_decision` | Decision | **排他的**．個人ごとの VOICE/SILENCE: ロジット (rule) または LLM 呼び出し．`voiced_last` を書き，チーム別の発言 / 援助 / エラー議論カウンタを加算する． |
| 3 | `learning_behavior_aggregate` | Interaction | カウンタから `L_k = w_v·V_k/n_k + w_h·H_k/n_k + w_e·E_k/n_k`． |
| 4 | `org_performance` | Reward | `Π_k = γ_L·L_k + γ_K·K_k + N(0, σ_obs²)`．知識ストック `K_k` (L の EMA) を減衰更新する．観察者ノイズはチーム別の凍結成分を含むため，tick 平均で消えない． |
| 5 | `psafety_update` | PostStep | **コア差分方程式**．ψ をスナップショットし，発言者の報復を抽選し，`ψ_i ← (1−λ)ψ_i + λ[α·s + β·c − γ·1[報復] + δ·ψ̄_{−i}] + disposition_i` を適用し一括書き戻して ψ̄_k を再計算する．個人 disposition (fear / ivt / concern) がチーム内 ψ 分散を有限に保ち，ICC(ψ) が 1 へ潰れず .39 付近に留まる． |
| 6 | `team_efficacy_update` | PostStep | η_k をチーム別の **独立** 潜在目標へ進化させ，効力感を ψ と弁別させる (H5/H8 の |t| < 2 検証)．ψ̄ が新しい状態を参照できるよう `psafety_update` の後に宣言する． |

宣言順 = 同一フェーズ内の発火順．Environment → Decision → Interaction → Reward → PostStep のフェーズ順序が原著の因果連鎖 (支援 → 発言 → 学習 → 業績 → 信念更新) を写し取る．

## 更新セマンティクス

判断メカニズムはステップ開始時に全個人をスナップショットし，スナップショットから新しい `voiced_last` を書く (同期更新．活性化順序に依存しない)．`psafety_update` も同様に旧スナップショットから全 ψ を計算してから書き戻す．

## 二層の決定論

- **決定論的 socsim コア** — `derive_seed(seed, &[0])` がワールド初期化，`&[1]` がエンジン (スケジューラ + rule-vote ベルヌーイ)，`&[2]` が報復抽選，`&[3]` が観察者ノイズをシードする．`rule` モードは bit 単位で再現する (`rule_mode_is_bit_deterministic` で検証)．
- **非決定的 LLM レイヤ** — `voice_decision` のみ．`socsim-llm` のプロンプト→応答キャッシュ + `temperature=0` + `(agent_id, t)` 由来シードで擬似決定論化する．

## メトリクス (`metrics.rs`)

ICC (一元配置分散分析．非平衡時の平均グループサイズ補正付き)・単/二変数 OLS・Baron & Kenny (1986) 三段階法は **論文固有** であり，ローカルに実装する．`socsim-metrics` からは `mean` / `variance` のみ流用する．
