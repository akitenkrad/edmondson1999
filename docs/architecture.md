# Architecture

## World state

`TeamWorld` holds:

- `individuals: BTreeMap<AgentId, Individual>` — sorted keys give a canonical, deterministic activation order. Each `Individual` carries `psi` (ψ_i ∈ [0,1]), `fear` (f_i), `ivt` (θ_i, implicit-voice-theory strength), `private_concern` (c_i), and the transient flags `voiced_last` / `retaliated_last`.
- `teams: BTreeMap<TeamId, Team>` — each `Team` carries its sorted `members`, the context `support` (s_k) and `coaching` (c_k), the aggregate `psi_bar` (ψ̄_k), `learning` (L_k), `performance` (Π_k), and `efficacy` (η_k).
- `network: SocialNetwork` — a single `socsim-net` graph holding **only within-team edges** (each team is wired by Watts–Strogatz independently; no cross-team edges exist).
- transient per-team counters `voice_count_by_team` / `help_count_by_team` / `error_talk_by_team`, cleared at the start of every step.

## Six mechanisms across the six-phase loop

| # | Mechanism | Phase | Role |
|---|-----------|-------|------|
| 1 | `context_support_update` | Environment | Clear the transient counters; mean-revert each team's support `s_k` / coaching `c_k` toward its target (optional exogenous shock). |
| 2 | `voice_decision_rule` / `voice_decision` | Decision | **Mutually exclusive.** Per-individual VOICE/SILENCE: a logit (rule) or an LLM call. Writes `voiced_last` and increments the per-team voice / help / error-talk counters. |
| 3 | `learning_behavior_aggregate` | Interaction | `L_k = w_v·V_k/n_k + w_h·H_k/n_k + w_e·E_k/n_k` from the counters. |
| 4 | `org_performance` | Reward | `Π_k = γ_L·L_k + γ_K·K_k + N(0, σ_obs²)`; decays the knowledge stock `K_k` (an EMA of L). The observer noise has a per-team frozen component so it does not average away over ticks. |
| 5 | `psafety_update` | PostStep | **Core diff-eq.** Snapshots ψ, draws retaliation for voicers, applies `ψ_i ← (1−λ)ψ_i + λ[α·s + β·c − γ·1[retal] + δ·ψ̄_{−i}] + disposition_i`, batch-writes, then recomputes ψ̄_k. The per-individual disposition (fear / ivt / concern) keeps within-team ψ variance finite, so ICC(ψ) stays near .39 instead of collapsing to 1. |
| 6 | `team_efficacy_update` | PostStep | Evolves η_k toward a per-team **independent** latent target, making efficacy discriminant from ψ (the H5/H8 |t| < 2 check). Declared after `psafety_update` so ψ̄ is fresh. |

Declaration order = firing order within a phase. The phase ordering Environment → Decision → Interaction → Reward → PostStep mirrors the paper's causal chain (support → voice → learning → performance → belief update).

## Update semantics

The decision mechanisms snapshot every individual at step start and write the new `voiced_last` from the snapshot (synchronous update; activation order does not matter). `psafety_update` likewise computes all new ψ from the old snapshot before writing.

## Two-layer determinism

- **Deterministic socsim core** — `derive_seed(seed, &[0])` seeds world init, `&[1]` the engine (scheduler + rule-vote Bernoulli), `&[2]` retaliation draws, `&[3]` observer noise. The `rule` mode reproduces bit-for-bit (asserted by `rule_mode_is_bit_deterministic`).
- **Non-deterministic LLM layer** — `voice_decision` only, pseudo-determinised by the `socsim-llm` prompt→response cache + `temperature=0` + `(agent_id, t)`-derived seed.

## Metrics (`metrics.rs`)

ICC (one-way ANOVA, with the unbalanced mean-group-size correction), simple/two-predictor OLS, and the Baron & Kenny (1986) three-step mediation are **paper-specific** and implemented locally. Only `mean` / `variance` are reused from `socsim-metrics`.
