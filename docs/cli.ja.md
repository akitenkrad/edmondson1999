# CLI リファレンス

`edmondson` バイナリは 3 つのサブコマンドを持つ．

## `run` — 単一構成

`--runs` 回の独立試行 (各々 `--seed` から派生した新しいシード) を実行し，最後の試行の long-format CSV と全試行をプールした `team_cross_section.csv` を書き出す．

| フラグ | 既定 | 意味 |
|------|------|------|
| `--decision-mode` | `rule` | `rule` (ロジット・LLM 不要) または `llm` (socsim-llm)． |
| `--n-teams` | `90` | チーム数． |
| `--team-size` | `8` | チームあたりの個人数． |
| `--network-model` | `watts-strogatz` | `watts-strogatz` / `erdos-renyi` / `barabasi-albert` (チーム内)． |
| `--network-k` | `6` | Watts–Strogatz / Barabási–Albert の次数パラメータ． |
| `--network-beta` | `0.15` | Watts–Strogatz の再配線 β (または Erdős–Rényi の p)． |
| `--lambda` | `0.10` | ψ 更新の学習率 λ． |
| `--alpha` | `0.30` | ψ 更新のコンテクスト支援重み α． |
| `--beta` | `0.25` | ψ 更新のコーチング重み β． |
| `--gamma` | `0.50` | ψ 更新の報復ショック重み γ． |
| `--delta` | `0.35` | ψ 更新の共有信念収束重み δ． |
| `--sigma-obs` | `0.22` | 観察者評定ノイズ sd σ_obs． |
| `--t-max` | `24` | 試行あたりのステップ数 (月相当)． |
| `--runs` | `30` | 独立試行数． |
| `--seed` | `1999` | ルートシード． |
| `--llm-temperature` | `0.0` | LLM 温度 (llm モード)． |
| `--llm-seed` | `0` | LLM シードオフセット (llm モード)． |
| `--cache-path` | `.llm_cache/cache.json` | プロンプト→応答キャッシュパス (llm モード)． |
| `--output-dir` | `results` | 出力ベースディレクトリ． |

## `sweep` — α × δ 感度分析

`α × δ` の直積でセルあたり `--runs` 試行を実行し，`sweep_summary.csv` に `(α, δ, run)` ごとに 1 行を書く．

| フラグ | 既定 | 意味 |
|------|------|------|
| `--alpha-min/max/step` | `0.10` / `0.50` / `0.10` | α 走査グリッド． |
| `--delta-min/max/step` | `0.10` / `0.60` / `0.10` | δ 走査グリッド． |
| `--lambda` | `0.10` | 走査中固定する λ． |
| `--n-teams` / `--team-size` | `90` / `8` | 組織形状． |
| `--runs` | `30` | セルあたり試行数． |
| `--t-max` / `--seed` | `24` / `1999` | 期間 / ルートシード． |

## `reproduce` — アンカーレポート

`--runs` 試行を実行し，各試行のチーム横断データで §5 アンカーを計算し，**試行平均** を PASS / off-anchor 判定とともに表示する (試行ごとの統計量を平均することで n≈51 チームの検定力を保ち，プールによる過剰検定力を避ける)．

| フラグ | 既定 | 意味 |
|------|------|------|
| `--decision-mode` | `rule` | レポート対象のモード． |
| `--n-teams` / `--team-size` | `90` / `8` | 組織形状． |
| `--t-max` / `--runs` / `--seed` | `24` / `30` / `1999` | 期間 / 試行数 / ルートシード． |
